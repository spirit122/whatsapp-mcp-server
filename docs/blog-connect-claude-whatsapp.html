How to Connect Claude with WhatsApp Business in 5 Minutes

Connecting Claude AI to WhatsApp Business opens up powerful automation capabilities: send messages, manage templates, handle media, and even receive incoming messages — all through natural language. This guide walks you through the entire setup using the WhatsApp MCP Server, from zero to sending your first message.

What Is an MCP Server for WhatsApp?

The Model Context Protocol (MCP) is an open standard that lets AI assistants like Claude interact with external services. The WhatsApp MCP Server acts as a bridge between Claude Desktop and the WhatsApp Business API, giving Claude access to 43 tools across 8 modules — messaging, templates, media, webhooks, analytics, and more.

Unlike traditional chatbot platforms, you don't need to write code or deploy anything. The server runs on Cloudflare Workers at the edge, so latency is under 50ms worldwide.

Prerequisites

  • Claude Desktop installed on your computer (Windows or Mac)
  • A Meta Business account with WhatsApp Business API access
  • A WhatsApp Business phone number registered in Meta Business Suite
  • Your WhatsApp Access Token from the Meta Developer Portal

Step 1: Open Claude Desktop Configuration

Claude Desktop uses a JSON configuration file to register MCP servers. You need to locate and edit this file.

On Windows

Open File Explorer and navigate to:

%APPDATA%\Claude\claude_desktop_config.json

On macOS

Open Finder and go to:

~/Library/Application Support/Claude/claude_desktop_config.json

Alternatively, open Claude Desktop, click the gear icon (Settings), go to Developer, and click Edit Config.

Step 2: Add the WhatsApp MCP Server

Paste the following JSON into your configuration file. If you already have other MCP servers configured, add the "whatsapp" entry inside the existing "mcpServers" object.

{
  "mcpServers": {
    "whatsapp": {
      "type": "url",
      "url": "https://whatsapp-mcp-server.eosspirit.workers.dev/mcp"
    }
  }
}
Tip: Make sure the JSON is valid. A missing comma or extra bracket will prevent Claude from loading the server. Use a JSON validator if unsure.

Step 3: Restart Claude Desktop

After saving the configuration file, fully quit Claude Desktop and reopen it. You should see the WhatsApp MCP Server listed in the tools panel. Claude now has access to all WhatsApp Business tools available on your plan.

Step 4: Authenticate with WhatsApp

The first time you use a WhatsApp tool, Claude will ask you to provide your WhatsApp Access Token and Phone Number ID. You can find these in the Meta Developer Portal under your WhatsApp app settings.

Once authenticated, your credentials are stored securely and you won't need to enter them again.

Step 5: Send Your First Message

Now the fun part. Simply ask Claude in natural language:

// Try any of these prompts:

"Send a WhatsApp message to +1 555 123 4567 saying
 your order #4521 is ready for pickup"

"Show me all my WhatsApp message templates"

"What's my WhatsApp business profile info?"

"Send an image to +1 555 987 6543 with the caption
 Here's your invoice for March"

Claude will use the appropriate MCP tool automatically. You'll see the tool call in the interface and the result from WhatsApp.

Troubleshooting Common Issues

Claude doesn't show the WhatsApp tools

Make sure you saved the config file correctly and restarted Claude Desktop completely (not just closed the window). Check that the JSON syntax is valid.

Authentication errors

Verify that your WhatsApp Access Token hasn't expired. Tokens from the Meta Developer Portal test mode expire after 24 hours. For production, generate a permanent token through your System User in Meta Business Suite.

Messages not delivering

WhatsApp Business API requires that the recipient has opted in to receive messages, or you must use an approved message template for the first contact. Regular text messages can only be sent within the 24-hour customer service window after the user messages you first.

Rate limit errors

The Free plan includes 100 requests per hour. If you're hitting limits, consider upgrading to Pro (1,000 req/hour) or Enterprise (10,000 req/hour).
